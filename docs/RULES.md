# NeverBroke 修改规则与全局计划

> **适用范围**：微信小程序「NeverBroke（认知翻身账）」全栈代码库。  
> **目的**：统一 UI 交互、代码风格与执行路径，降低改错、串页、样式碎片化风险。

---

## 一、UI 交互规则（优先级最高）

### 1.1 横向选项控件（分段/Chip）

- **适用场景**：选项数量 ≤ 5 且需要横向滑动的类型/币种/状态筛选。  
- **实现**：使用 `scroll-view` + 白底描边 `type-group` / `type-btn`；选中项绿色高亮（`#1aad19`）。
- **不使用**：单选 `radio` 竖排、长列表 `picker`（选项多且需手写时除外）。
- **复用**：账户类型、币种、电子钱包机构、账户列表筛选等，保持视觉语言一致。

### 1.2 表单显隐（按类型/条件）

- **字段显隐**：严格按业务类型控制，不要「所有类型同一套表单」。  
  - 现金：无机构、无尾号。  
  - 电子钱包：横向机构 + 可选「其他」说明；无尾号。  
  - 银行卡/信用卡：机构 + 可选卡号尾号。  
- **文案区分**：  
  - 信用卡：**「初始余额 / 已用金额」** + 灰色说明。  
  - 其他：仅 **「初始余额」**，不写「/ 已用金额」。

### 1.3 编辑态与新建态

- **类型不可改**：编辑时类型按钮 `disabled`（降低透明度 + 点击无效）。  
- **标题**：区分引导/新建/编辑标题（`navigationBarTitle`）。  
- **数据重置**：每次进入新建（无 `id`）在 `onLoad` 里整表默认化，避免页面实例复用导致数据串页。

### 1.4 金额输入防串数据（UI 层）

- 使用 **`amount-input`** 时，用 **`wx:for="{{[formKey]}}"`** 包一层，依赖 **`formKey` 递增** 强制重建组件，避免原生 `input` 残留显示。  
- **每次进入新建**（无 `id`）在 `onLoad` 里整表默认化并 **`formKey++`**；**创建成功**离开前再重置一次。

## 二、代码风格与兼容性

### 2.1 避免触发 Babel runtime

- **不使用** 对象/数组展开 `...`、`[...arr]`、`...(condition ? obj : {})` 等写法，改用 **`Object.assign` / `.slice()` / 分支赋值**，避免真机报 `@babel/runtime/helpers/objectSpread2`。

### 2.2 云函数（Node.js）

- **展示名不落库**：`list` / `get` 返回时即时拼 `name`，写入时不存 `name` 字段。  
- **去重查询**：用单字段（`openid`）拉列表 + 内存判断，避免复合查询缺索引导致报错。  
- **对象合并**：用 `Object.assign` 代替对象展开，避免个别运行环境兼容问题。

## 三、全局修改计划

### 3.1 近期（当前已完成）

- [x] **账户体系**：四类账户（现金/银行卡/电子钱包/信用卡）、机构/尾号/币种、展示名不落库、现金去重查询兼容。  
- [x] **UI 控件**：类型、币种、电子钱包机构横向分段；账户列表类型快捷筛选。  
- [x] **金额与文案**：按类型区分余额文案；`formatMoney` 支持多币种符号。  
- [x] **兼容性**：去掉对象展开；现金去重改为内存判断；`withDisplayName` 改用 `Object.assign`。

### 3.2 短期（下一步建议）

- [ ] **「记一笔」页**：账户选择、分类选择、快捷金额等与账户体系对齐（确保列表里账户名与展示名一致）。  
- [ ] **转账页**：确认使用 `fromAccountId`，与云函数规则一致。  
- [ ] **报表/现金流/时间线**：确保多币种时总资产显示为「多币种」，不进行错误折算。  
- [ ] **设置页**：语言切换、隐藏金额、订阅消息等按 `settings.js` 现有逻辑，无需大改。

### 3.3 中期（UI 统一与体验）

- [ ] **统一 Chip/Tab 样式**：将 `type-group` / `type-btn` 抽取为全局组件，减少重复样式。  
- [ ] **表单页**：预算设置、分期设置、周期收入设置等，复用横向控件与表单重置规则。  
- [ ] **空态与引导**：首页无账户、账本无数据等空态与引导跳转，保持视觉与文案一致。

### 3.4 长期（功能扩展与优化）

- [ ] **多设备同步**：登录/会话管理、跨设备数据同步策略。  
- [ ] **数据导出**：支持导出为 CSV/Excel，并按账户、币种分别展示。  
- [ ] **报表增强**：多币种报表切换、汇率管理（如需）。

## 四、执行顺序与依赖

1. **账户体系**（已完成）→ 2. **记一笔/账本/转账**（对齐账户展示名与类型）→ 3. **报表/现金流/时间线**（多币种总资产显示）→ 4. **设置页**（语言/隐藏金额等）→ 5. **UI 组件统一**（抽取全局 Chip/Tab）→ 6. **功能扩展**（多设备同步、数据导出等）。

---

## 五、注意事项

- **向后兼容**：旧数据中的 `savings` / `investment` 等类型仍可编辑与展示，不做数据迁移。  
- **云函数部署**：修改云函数后需重新上传并部署，否则小程序端仍调用旧逻辑。  
- **测试覆盖**：每次改动后，建议在真机与开发者工具各跑一遍核心流程（登录 → 新建账户 → 记账 → 报表）。
