# 人工任务执行指令清单（NeverBroke）

生成时间：2026-04-10

> 本文档整理了当前仓库内无法通过代码自动完成、必须由人工在微信公众平台、云开发控制台、真机或开发者工具中执行的事项。建议按优先级顺序执行，并在完成后同步更新 `docs/PENDING_TASKS.md`、`docs/TESTING.md`、`docs/TEST_REPORT.md`。

## 一、云开发控制台

### 1. 上传并部署全部生产所需云函数
**执行平台：** 微信开发者工具 / 云开发控制台

**步骤：**
1. 打开微信开发者工具，进入当前小程序项目。
2. 确认项目绑定的是正式云环境。
3. 进入云开发面板，找到所有生产所需云函数。
4. 逐个执行“上传并部署：云端安装依赖”。
5. 重点确认：`login`、`account`、`transaction`、`budget`、`report`、`cashflow`、`dataManage`、`settings`、`notify`。

**预期结果：**
- 全部云函数部署成功，无报错。

**如何验证：**
- 云函数列表中可看到最新部署时间。
- 前端调用相关云函数时返回正常。

### 2. 创建数据库集合
**执行平台：** 云开发控制台

**步骤：**
1. 进入目标云环境的数据库页面。
2. 创建以下集合：
   - `users`
   - `accounts`
   - `transactions`
   - `budgets`
   - `installment_plans`
   - `recurring_incomes`
   - `user_settings`
   - `salary_plans`
   - `notification_log`

**预期结果：**
- 所有集合创建完成，名称无误。

**如何验证：**
- 集合列表中能看到全部 9 个集合。
- 云函数访问不再报“集合不存在”。

### 3. 创建数据库索引
**执行平台：** 云开发控制台

**步骤：**
1. 进入各集合的索引管理。
2. 创建以下索引：
   - `transactions` → `openid + date`
   - `budgets` → `openid + year + month`
   - `accounts` → `openid`
   - `installment_plans` → `openid + status`
   - `recurring_incomes` → `openid + nextDueDate`
   - `salary_plans` → `openid`
   - `user_settings` → `openid`
   - `notification_log` → `openid + type + date`

**预期结果：**
- 关键查询路径具备对应索引。

**如何验证：**
- 索引列表中可看到目标索引。
- 相关查询无性能或索引缺失问题。

### 4. 检查数据库权限与数据隔离
**执行平台：** 云开发控制台

**步骤：**
1. 检查各集合权限规则。
2. 确保不同用户无法读写彼此的数据。
3. 如需客户端直连，至少设置为“仅创建者可读写”。
4. 用两个测试账号验证数据隔离。

**预期结果：**
- 数据权限符合最小权限原则。

**如何验证：**
- A 账号看不到 B 账号数据。
- 敏感集合无越权访问风险。

## 二、订阅消息与后台配置

### 5. 创建并配置订阅消息模板
**执行平台：** 微信公众平台

**步骤：**
1. 登录微信公众平台。
2. 进入“小程序 → 订阅消息”。
3. 创建并审核以下模板：
   - 预算提醒
   - 分期还款提醒
   - 周期收入提醒
4. 记录每个模板的模板 ID。
5. 将模板 ID 填入：
   - `miniprogram/config/subscribe.js` 的 `TEMPLATE_IDS`
   - `miniprogram/config/subscribe.js` 的 `SERVER_TEMPLATE_IDS`

**预期结果：**
- 三类模板均可用，并已有模板 ID。

**如何验证：**
- 后台模板列表中可见对应模板。
- 代码配置文件中已填入实际 ID。

### 6. 校验模板类目与业务场景一致
**执行平台：** 微信公众平台

**步骤：**
1. 查看模板详情。
2. 检查模板类目、字段是否匹配预算、分期、周期收入场景。
3. 如不合适，重新选择/申请模板。

**预期结果：**
- 模板用途与产品业务一致。

**如何验证：**
- 审核时可合理解释触发场景与发送内容。

### 7. 配置 `notify` 定时触发器
**执行平台：** 云开发控制台

**步骤：**
1. 确认 `notify` 云函数已部署。
2. 进入触发器管理。
3. 创建定时触发器并设置执行时间。
4. 启用触发器。

**预期结果：**
- `notify` 可按计划自动执行。

**如何验证：**
- 触发器列表中存在启用中的定时任务。
- 到时间后日志里有执行记录。

### 8. 核对发送日志与发送对象
**执行平台：** 云开发控制台 + 真机

**步骤：**
1. 用一个已授权账号、一个未授权账号测试。
2. 制造触发条件或手动触发 `notify`。
3. 查看 `notification_log` 与云函数日志。
4. 确保只给已授权用户发送。

**预期结果：**
- 已授权用户可收到消息，未授权用户不会收到。

**如何验证：**
- 收件结果与日志一致。
- 失败原因可追踪。

## 建议执行顺序
1. 云开发：部署云函数、建库、建索引、检查权限
2. 微信后台：创建订阅消息模板并填写模板 ID
3. 服务端提醒：配置 `notify` 定时触发器并验证日志
4. 真机/开发者工具：跑核心功能回归
5. 审核材料：协议、隐私、类目、素材
